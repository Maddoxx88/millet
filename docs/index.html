Semantic HTML, styled.

Millet is a tiny semantic-first CSS layer for common UI elements. Build with native tags and optional progressive enhancement, without framework lock-in.

Features

Light like an oat flake

6KB CSS, 2.2KB JS, minified + gzipped.

That's it.

Zero dependencies

Fully-standalone with no dependencies on any JS or CSS frameworks or libraries. No Node.js ecosystem garbage or bloat.

Semantic HTML

Native elements like <button>, <input>, <dialog> and semantic attributes like role="button" are styled directly. No classes.

Accessibility

Semantic HTML and ARIA roles are used throughout, with strong keyboard navigation support for components and elements.

Easy customization

Easily customize the overall theme by overriding a handful of CSS variables. Includes built-in light and dark themes.

Why?

Building UI often means choosing between heavy component frameworks and unstructured utility classes. Millet fills the middle ground: semantic markup first, low setup cost, and sensible defaults that remain easy to override.

About the author

Millet is created by Sunit Shirke to provide a practical UI baseline for teams who want speed and consistency while still writing semantic HTML.
